# 设计文档 · 气运抽签「按愿望内容加权」

> 版本：v0.2（详细设计稿）　适用：《骰它》气运抽签模块　约束：**纯离线 / 零依赖 / 即时**
> 一句话目标：让成功率**随用户写的愿望内容产生倾向**——越日常、越易实现 → 越容易摇高；越浮夸、越离谱、越贪心 → 越容易摇低。`我想吃饭` 偏高，`我想毁灭世界` 偏低；同时**保留玄学的随机惊喜**。

## 1. 背景与目标

现状：气运抽签对任何输入都是 `1~100` 的均匀随机。用户希望它"读懂"愿望的**分量**，给出有倾向的成功率，增加趣味与代入感、提升分享欲（结果卡片更有梗）。

**目标**
- G1：内容能影响成功率的**期望值**（不是写死结果）。
- G2：**100% 离线**，进 APK 不联网、无 AI、无第三方服务。
- G3：**即时**（<1ms 级），不卡顿。
- G4：**可玩、可扩展、可关闭**。

**非目标（明确不做）**
- N1：不追求"真正理解语义"，只做模式匹配（够用即可）。
- N2：不做道德/政治判官，不评价价值观。
- N3：不做严肃情绪识别/心理干预（仅在敏感词上做"不娱乐化"的保护性降级）。
- N4：不依赖中文分词器（体积/复杂度不划算），用子串与词表匹配。

## 2. 设计原则

| 原则 | 说明 |
|------|------|
| 离线优先 | 词表 + 规则全部内置，纯前端 JS。 |
| 趣味优先于准确 | 这是玄学小游戏，"偶尔猜不到"也是梗，不追求严谨。 |
| 偏向而非写死 | 内容只决定**分布的均值**，结果仍随机，永远保留爆冷/爆好。 |
| 公平兜底 | 空输入 / 未命中任何词 → **退回纯随机**，不冤枉用户。 |
| 中立无害 | 不碰敏感价值判断；危险内容做保护性降级。 |
| 可解释但留悬念 | 极端情况给一句俏皮旁白，但不暴露具体算法，保留玄学感。 |
| 可关可扩 | 「玄学加权」开关（默认开）；词表易扩充。 |

## 3. 总体架构与数据流

```
用户输入 text
   │
   ▼  ① 归一化 normalize(text)         → 统一大小写/全半角/去标点
   │
   ▼  ② 特征提取 extractFeatures()      → 命中词、程度词、否定、长度、数量级…
   │
   ▼  ③ 评分 scoreToMean()             → 目标均值 mean ∈ [μ_min, μ_max]
   │
   ▼  ④ 采样 sampleAroundMean(mean)    → 三角分布抽 pct ∈ [1,100]
   │
   ▼  ⑤ 彩蛋/文案 flavor(mean,pct)      → 可选旁白
   │
   ▼  结果：pct(%) + 可选 flavor
```
开关关闭时：直接 `pct = 1 + floor(rand()*100)`（现有逻辑），跳过 ①~⑤。

---

## 4. 文本预处理与归一化

`normalize(text)`：
1. 截断到合理长度（如 ≤ 40 字，超出只取前 40，防极端输入）。
2. 全角 → 半角（数字/字母/标点）。
3. 英文统一小写。
4. 去除空白与标点（保留中文、字母、数字）。
5. （可选）繁体 → 简体的小映射表，覆盖常见高频字（毀滅→毁灭 等）；不做全量。
6. 输出 `norm` 字符串，供子串匹配。

> 不做分词：直接用 `norm.includes(词)` 做子串命中（中文词典子串匹配召回足够；个别误命中用"长词优先 + 词边界白名单"缓解）。

### 5.2 权重规则
- **长词优先**：先匹配更长的词（`一夜暴富` 命中则不再单独叠 `暴富`/`一夜`），避免重复计分。
- **每类封顶**：同一标签（daily/greed/…）累计权重设上限（如 ±45），防堆砌。
- **多词叠加递减**：第 2 个同类词权重 ×0.6，第 3 个 ×0.3（边际递减）。

### 5.3 放大词（程度副词，作乘数而非加数）
```js
const AMP = ['全世界','所有人','全人类','整个','永远','一定','必须','立刻','马上','彻底','瞬间','宇宙'];
// 命中任一 → 把"当前净偏移"再乘 1.3（把大愿望推得更极端、把日常也略放大）
```

## 6. 启发式特征规则（词表之外的兜底）

即使没命中词，也能从"句子形态"判断分量：

| 特征 | 判定 | 影响 |
|------|------|------|
| 宏大度 | 含「全/所有/世界/宇宙/人类/永远」 | 均值 −（叠加 AMP） |
| 数量级 | 含「亿/万亿/几千万/无数」 | 均值 − |
| 努力信号 | 含「努力/坚持/复习/练习/加班」 | 均值 +（天道酬勤） |
| 可行性 | 含「今天/明天/这次/一点点/稍微」 | 均值 +（目标具体） |
| 长度 | 文本越长越"宏愿"倾向 | 每超 12 字 −2，封顶 −10 |
| 感叹强度 | `！` 数量多 | 轻微 −（野心外溢，象征性） |

## 7. 否定 / 反讽 / 程度词处理

- **否定翻转**：命中"升运词"前若紧邻否定词（不/别/没/不想/不要/拒绝），**翻转该词符号**。
  例：`不想上班` → "上班"本中性，"不想"使其略升（摸鱼倾向）；`别生病` → "生病"负，被否定→转正（祝愿成立）。
- **否定词表**：`['不','别','没','无','勿','拒绝','不想','不要','不会','再也不']`，作用范围：否定词后 3~4 字内。
- **反讽**：不专门识别（成本高、易错）。接受偶发"看走眼"，当玄学梗。
- 程度词见 §5.3（放大）。

> 实现上：先做否定标记，再算词权重，最后乘程度放大。

## 8. 评分模型（内容 → 目标均值）

```
raw = 50                          // 中性基准
raw += Σ 命中词权重（经长词优先、每类封顶、递减、否定翻转后）
raw += Σ 启发式特征偏移
if 命中放大词: raw = 50 + (raw-50) * 1.3
mean = clamp(raw, μ_min, μ_max)   // 夹到均值区间
```
- 推荐 `μ_min = 12`，`μ_max = 88`（永远给点希望/悬念，不到 0/100 的"确定"）。
- 若**没有命中任何词、也没触发任何特征** → 标记 `neutral=true`，走纯随机（见 §12）。

## 9. 概率采样（均值 → 最终百分比）

**为什么不直接用均值当结果？** 那样就成了"查字典报数"，失去玄学随机。要的是**以内容为中心的概率分布**。

采用**三角分布**（简单、可控、形状自然）：
```
function sampleAroundMean(mean, spread=34){
  // 三角分布：两个均匀随机之和的对称特性，围绕 mean
  const r = (rand()+rand())/2;            // ∈[0,1]，峰在 0.5
  let v = mean + (r-0.5)*2*spread;        // 以 mean 为中心，±spread
  return clamp(Math.round(v), 1, 100);
}
```
- `spread` 控制"运气波动幅度"，推荐 **34**（够随机、又看得出倾向）。
- 边缘 `mean` 靠近上下限时，分布自然被 `[1,100]` 截断（合理）。
- 想要更"重尾"（更容易爆冷/爆好）可用 `(rand()*rand())` 偏态或混入 5% 纯均匀（见 §10）。

**每日变化**：天然随机即可；无需种子。若想"同一句话当天多次结果一致"——但同日同事本就只问一次（§20），无需处理。

## 10. 极值、彩蛋与文案

- **保底爆冷/爆好**：采样后以 **5% 概率**改用纯均匀 `1~100`，让"毁灭世界"也有渺茫的高光、"吃饭"也可能翻车——强化玄学不可测。
- **极值文案（可选，低调）**：仅在 `mean` 很低或很高时，在结果下方加一句小字旁白，**不解释算法**：
  - `mean ≤ 20`：`📜 天机似乎不太看好这桩宏愿……`
  - `mean ≥ 80`：`📜 顺天而行，签面颇为和善。`
  - 命中 absurd 类且摇出高分：`📜 离谱的愿望撞上离谱的好运。`
- 文案池可扩；默认**可在设置里关**（保持神秘感）。

## 11. 反作弊与可玩性平衡

| 手段 | 目的 |
|------|------|
| 同类递减（§5.2） | 防"吃饭吃饭吃饭"堆词刷高。 |
| 每类封顶 | 单一倾向不过载。 |
| 均值夹紧 12~88 | 永不 100% 确定，留悬念。 |
| 5% 纯随机保底 | 不可被完全预测。 |
| 同日同事只问一次（现有） | 同句不可当天反复刷。 |

> 立场：**"写吃饭就高"本身是彩蛋乐趣**，不必强防；以上仅防极端堆砌导致体验失衡。

## 12. 边界情况与默认行为

| 输入 | 行为 |
|------|------|
| 空字符串 | 纯随机 `1~100`（同现状）。 |
| 未命中任何词且无特征 | `neutral` → 纯随机（公平兜底）。 |
| 纯数字 / 纯英文 / emoji | 走特征规则；命不中 → 纯随机。 |
| 超长文本 | 截断 40 字 + 长度特征轻微降。 |
| 含敏感/危险词 | 见 §13，**不娱乐化**。 |

## 13. 安全 / 伦理 / 隐私

- **不做价值审判**：词表只玩"大小 / 难易 / 贪心 / 浮夸"维度；**不收录政治、宗教、种族、性别等敏感判断**。
- **危险内容保护性降级**：若命中自伤 / 自杀 / 伤害他人类词（内置一个**小型保护词表**），**不参与玄学加权**——直接走中性随机，且**不显示任何嘲讽/极端文案**；可选弹一句温和提示（如「遇到困难记得找人聊聊」）。宁可"不好玩"也不冒犯。
- **隐私**：所有输入与判定**全在本地**，不上传、不联网；愿望文本只存在 `localStorage`（备份码导出时包含，属用户自有数据）。
- **可移除**：词表与逻辑均可被「玄学加权」开关整体关闭。

## 14. UI 与开关设计

- **开关「玄学加权」**：布尔，存于 `save.settings.luckBias`，**默认 true**。
  - 放置：数据档案页「🗄️ 数据管理」附近，或在气运输入框旁加一个小标签可切换。
  - 关闭后：气运完全回到纯 `1~100` 随机。
- **旁白开关「天机旁白」**：布尔，默认 true，控制 §10 文案显示。
- **结果卡片（#10）**：分享卡片照常显示百分比；可选附上旁白小字。
- **不暴露分值**：界面不显示"因为命中XX所以降权"，保留玄学感（开发期可加隐藏调试开关）。

## 16. 参考实现（伪代码，约 60~90 行）

```js
const NEG = ['不','别','没','无','勿','不想','不要','不会','再也不','拒绝'];
const AMP = ['全世界','所有人','全人类','整个','永远','一定','必须','立刻','马上','彻底','瞬间','宇宙','无数'];
const SAFE_BLOCK = [/* 自伤/伤害类小词表：命中即降级，不娱乐化 */];

function normalize(t){
  t = (t||'').slice(0,40).toLowerCase();
  // 全角转半角 + 去标点空白（实现略）
  return t.replace(/[\s\p{P}]/gu,'');
}

function luckRoll(text){
  if(!settingsOn('luckBias')) return { pct: 1+Math.floor(rand()*100), flavor:null };
  const norm = normalize(text);
  if(!norm) return { pct: 1+Math.floor(rand()*100), flavor:null };

  if(SAFE_BLOCK.some(w=>norm.includes(w)))                       // §13 保护
    return { pct: 1+Math.floor(rand()*100), flavor:'遇到难处，记得找人聊聊。' };

  // —— 词命中（长词优先、否定翻转、同类递减、封顶）——
  let raw = 50, used = new Set(), tagSum = {}, tags = [];
  for(const [w,wt,tag] of LUCK_LEX.slice().sort((a,b)=>b[0].length-a[0].length)){
    if(!norm.includes(w) || used.has(w)) continue;
    used.add(w);
    let sign = 1;
    const idx = norm.indexOf(w);
    const pre = norm.slice(Math.max(0,idx-4), idx);
    if(NEG.some(n=>pre.includes(n))) sign = -1;                  // 否定翻转
    const n = (tagSum[tag]||0); const decay = n===0?1:(n===1?0.6:0.3);
    let add = sign*wt*decay;
    raw += add; tagSum[tag] = n+1; tags.push(tag);
  }
  // —— 启发式特征（略：宏大/数量级/努力/可行/长度）——
  raw += featureBias(norm);
  const neutral = (tags.length===0) && featureBias(norm)===0;
  if(neutral) return { pct: 1+Math.floor(rand()*100), flavor:null };

  if(AMP.some(a=>norm.includes(a))) raw = 50 + (raw-50)*1.3;     // 程度放大
  const mean = clamp(raw, 12, 88);

  // —— 采样（5% 纯随机保底）——
  let pct;
  if(rand()<0.05) pct = 1+Math.floor(rand()*100);
  else { const r=(rand()+rand())/2; pct = clamp(Math.round(mean+(r-0.5)*2*34),1,100); }

  return { pct, flavor: settingsOn('luckFlavor') ? flavor(mean,pct,tags) : null };
}
```

> 接入点：把现有 `roll()` 里 `current==='luck'` 分支的 `const n=1+Math.floor(Math.random()*100)` 换成 `const {pct:n, flavor}=luckRoll(ev)`，并在结果区追加 `flavor`。其余（飞签动画、同日只问一次、成就统计）不变。

## 19. 性能与体积

- 词表 ~150–400 条，纯文本约 **3–8 KB**，对 APK 体积可忽略。
- 匹配为 O(词数 × 文本长度)，文本 ≤40 字、词数百级，单次 **<1ms**。
- 可选优化：构建首字索引或正则联合，进一步降常数（通常没必要）。

## 20. 与现有系统的关系

- **同日同事只问一次**：保持；加权只改"取值"，不改这条规则。
- **成就统计**：`ev_luck1/50/100`、高低运计数照常按最终 `pct` 触发——加权后"摇到 1%/100%"更难/更易，正好让相关成就更有故事性。可考虑新增成就：`【离谱的愿望】许了个 absurd 类愿望`、`【天道酬勤】effort 类摇出 ≥80`。
- **结果卡片（#10）**：展示最终 `pct`（+可选旁白），分享更有梗。
- **备份码（#1）**：`save.settings` 一并导出导入。
- **数据可视化（#9）**：气运分布图会因加权而呈现倾向，属预期。

## 22. 风险与缓解

| 风险 | 影响 | 缓解 |
|------|------|------|
| 词典覆盖有限 | 很多愿望走纯随机，"不够聪明" | 公平兜底本就合理；持续扩词；不命中不乱给 |
| 被说"不公平/作弊" | 部分用户只想纯随机 | 默认可关的开关；保底 5% 纯随机；不暴露分值 |
| 子串误命中 | 个别词被错配 | 长词优先 + 关键词加边界白名单 |
| 否定漏判 | "不想"类偶尔判反 | 接受偶发，当玄学梗；范围窗口可调 |
| 敏感/危险内容 | 冒犯或不当娱乐化 | 保护词表降级 + 温和提示 + 不显极端文案 |
| 维护成本 | 词表需长期更新 | 词表与逻辑解耦，改词不改码 |
